# AGENT INSTRUCTIONS

Цей файл діє для всього репозиторію `web-scraper-generator` і описує локальні правила, які потрібно виконувати під час редагування коду.

## Як орієнтуватися в проєкті
- Перед змінами перегляньте `README.md` та `project_structure.md`, щоб розуміти основні модулі.
- Якщо додаєте або змінюєте функціонал, обов'язково описуйте його у відповідних README/довідкових файлах, лише якщо зміни впливають на спосіб запуску чи архітектуру.

## Стиль коду
- Всі нові коментарі, docstring-и та повідомлення логів пишіть українською з короткими та зрозумілими формулюваннями.
- Дотримуйтеся PEP8 (довжина рядка до 100 символів, зрозумілі імена змінних, типізація, якщо вона вже використовується).
- Не використовуйте `try/except` навколо імпортів.

## Тести та запуск
- Після змін, які впливають на генерацію/валідацію скрейперів, запускайте релевантні CLI-команди або pytest-тести, якщо це можливо без зовнішніх секретів.
- Якщо команду неможливо виконати (нема API ключа тощо), поясніть причину у фінальному повідомленні.

## Документація AGENT
- Якщо створюєте додаткові папки з окремими правилами, додавайте туди власні `AGENTS.md`, які уточнюють ці інструкції.
